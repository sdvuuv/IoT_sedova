### Задача:

Написать прошивку, которая позволяет откалибровать движение колёсной платформы без её перепрошивки. Калибровочные команды поступают по Bluetooth-каналу: модуль работает как UART по воздуху и подключён к аппаратному Serial. От gamepad-приложения при нажатии кнопок X, B, A, Y и крестовины <, >, ^, v отправляется определённый символ, и прошивка реагирует на него соответствующим образом. Подобранные значения сохраняются в EEPROM, чтобы не сбрасываться после выключения питания.

### Требования:

- Связь по Bluetooth через аппаратный Serial, без проводного подключения; питание автономное.
- Драйвер моторов L298N: каждый мотор управляется парой входов IN1/IN2 (направление) и пином EN (скорость через ШИМ).
- Выбор настраиваемой стороны платформы.
- Подбор направления вращения каждого мотора (провода мотора могут быть подключены двумя способами).
- Балансировка скоростей левого и правого колеса, чтобы движение было прямолинейным.
- Калибровка времени поворота на месте отдельно для 90, 180, 270 и 360 градусов, так как из-за инерции и проскальзывания время поворота на 360 не равно учетверённому времени поворота на 90.
- Хранение калибровки в EEPROM с восстановлением после включения.
- Перебор четырёх комбинаций HIGH/LOW для пары IN-входов и закрепление подобранной комбинации как направления «вперёд».

### Рассмотрим код и логику его работы.

##### 1. Подключение библиотеки и пины драйвера

```cpp
#include <EEPROM.h>

const uint8_t LEFT_EN  = 6;
const uint8_t LEFT_IN1 = 8;
const uint8_t LEFT_IN2 = 9;

const uint8_t RIGHT_EN  = 5;
const uint8_t RIGHT_IN1 = 11;
const uint8_t RIGHT_IN2 = 10;
```

- `EEPROM` используется для энергонезависимого хранения подобранной калибровки.
- Каждый мотор драйвера L298N управляется тройкой пинов: `IN1` и `IN2` задают направление вращения, `EN` — скорость через ШИМ.
- Пары пинов вынесены в именованные константы, чтобы код управления моторами не зависел от конкретной разводки.
- Пины 0 и 1 заняты аппаратным Serial — именно к ним подключается Bluetooth-модуль, поэтому отдельный программный порт не нужен.

##### 2. Коды команд gamepad

```cpp
#define CMD_X   'X'
#define CMD_B   'B'
#define CMD_A   'A'
#define CMD_Y   'Y'
#define CMD_LT  '<'
#define CMD_RT  '>'
#define CMD_UP  '^'
#define CMD_DN  'v'
```

- Каждой кнопке gamepad-приложения сопоставлен символ, который Bluetooth-модуль присылает по UART.
- X выбирает настраиваемую сторону, B инвертирует направление мотора, Y перебирает комбинации HIGH/LOW, A фиксирует выбранную комбинацию как «вперёд».
- Крестовина < и > изменяет числовое значение текущего параметра, ^ и v переключают, какой именно параметр редактируется.
- Дополнительно заданы служебные символы для проверочных заездов, сохранения, сброса и вывода состояния — они не относятся к gamepad, но полезны при отладке.

##### 3. Структура калибровки

```cpp
struct Calibration {
  uint16_t magic;
  bool     invertLeft;
  bool     invertRight;
  uint8_t  speedLeft;
  uint8_t  speedRight;
  uint16_t turnMs[ANGLE_COUNT];
};
```

- В одной структуре собрано всё, что подбирает пользователь и что целиком записывается в EEPROM одной операцией.
- `magic` — метка валидности: при чтении из EEPROM по ней проверяется, что там действительно лежит наша калибровка, а не мусор.
- `invertLeft` и `invertRight` — признаки инверсии каждого мотора; инверсия меняет местами IN1 и IN2 и тем самым учитывает способ подключения проводов мотора.
- `speedLeft` и `speedRight` задают раздельные скорости моторов для балансировки прямолинейного хода.
- `turnMs` — массив из четырёх значений времени поворота, по одному на каждый калибруемый угол.

##### 4. Комбинации HIGH/LOW

```cpp
const Combo COMBOS[4] = {
  { LOW,  LOW  },
  { HIGH, LOW  },
  { LOW,  HIGH },
  { HIGH, HIGH }
};
```

- Это четыре возможные комбинации сигналов на паре входов IN1/IN2 одного мотора.
- Комбинации (LOW, LOW) и (HIGH, HIGH) мотор не вращают, а (HIGH, LOW) и (LOW, HIGH) задают два противоположных направления.
- Кнопка Y последовательно перебирает индекс в этом массиве, кнопка A закрепляет текущую комбинацию как направление «вперёд».
- Такой перебор повторяет описанный в задании алгоритм калибровки: случайно выбирается состояние, проверяется результат движения, и при необходимости комбинация переключается на следующую.

##### 5. Работа с EEPROM

```cpp
void saveCalibration() {
  EEPROM.put(EEPROM_ADDR, calib);
}

void loadCalibration() {
  Calibration tmp;
  EEPROM.get(EEPROM_ADDR, tmp);
  if (tmp.magic == CALIB_MAGIC) calib = tmp;
  else                          loadDefaults();
}
```

- `saveCalibration` записывает всю структуру в EEPROM по команде сохранения.
- `loadCalibration` читает структуру при включении и проверяет метку `magic`: если метка совпала, калибровка восстанавливается, иначе подгружаются значения по умолчанию.
- За счёт этого подобранные настройки переживают выключение питания, и перепрошивка платы не требуется.

##### 6. Управление моторами L298N

```cpp
void setMotor(uint8_t en, uint8_t in1, uint8_t in2, int speed, bool inverted) {
  if (inverted) speed = -speed;
  speed = constrain(speed, -255, 255);
  if (speed > 0) {
    digitalWrite(in1, HIGH); digitalWrite(in2, LOW);  analogWrite(en, speed);
  } else if (speed < 0) {
    digitalWrite(in1, LOW);  digitalWrite(in2, HIGH); analogWrite(en, -speed);
  } else {
    digitalWrite(in1, LOW);  digitalWrite(in2, LOW);  analogWrite(en, 0);
  }
}
```

- `setMotor` — единственная низкоуровневая функция, напрямую обращающаяся к пинам драйвера; остальные движения построены поверх неё.
- Скорость передаётся со знаком: знак задаёт направление вращения, модуль — величину ШИМ; флаг `inverted` переставляет IN1 и IN2 местами для учёта разводки моторов.
- `setLeftMotor` и `setRightMotor` подставляют пины конкретного мотора и его признак инверсии из калибровки.
- `driveForward` и `driveBackward` задают обоим моторам скорость с нужным знаком; `rotateLeftRaw` и `rotateRightRaw` крутят моторы в противоположные стороны для разворота на месте.
- `turnByAngle` выполняет разворот и держит его ровно `turnMs` миллисекунд для нужного угла, после чего останавливает моторы.

##### 7. Закрепление комбинации как «вперёд»

```cpp
void applyComboAsForward() {
  Combo c = COMBOS[comboIndex];
  bool needInvert;
  if (c.in1 == HIGH && c.in2 == LOW)      needInvert = false;
  else if (c.in1 == LOW && c.in2 == HIGH) needInvert = true;
  else { /* комбинация не вращает мотор */ return; }
  if (currentSide == SIDE_LEFT) calib.invertLeft  = needInvert;
  else                          calib.invertRight = needInvert;
}
```

- Функция вызывается кнопкой A и связывает выбранную перебором комбинацию IN1/IN2 с направлением «вперёд».
- В схеме `setMotor` направлению «вперёд» соответствует пара IN1=HIGH, IN2=LOW; если перебираемая комбинация совпала с ней, инверсия не нужна, если зеркальная — мотор помечается как инвертированный.
- Комбинации без вращения отбрасываются с предупреждением, чтобы пользователь выбрал рабочую пару.

##### 8. Изменение параметров крестовиной

```cpp
void changeParam(int delta) {
  switch (currentParam) {
    case P_SPEED_LEFT:
      calib.speedLeft = constrain((int)calib.speedLeft + delta * 5, 0, 255);
      break;
    ...
  }
}
```

- `currentParam` указывает, какой параметр сейчас редактируется; кнопки ^ и v переключают его по кругу.
- `delta` равен +1 для кнопки > и -1 для кнопки <; скорость меняется шагом 5, время поворота — шагом 25 мс.
- `constrain` удерживает значения в допустимых пределах, чтобы калибровка не вышла за границы диапазона.

### setup()

```cpp
void setup() {
  pinMode(LEFT_EN,  OUTPUT);
  pinMode(LEFT_IN1, OUTPUT);
  pinMode(LEFT_IN2, OUTPUT);
  pinMode(RIGHT_EN,  OUTPUT);
  pinMode(RIGHT_IN1, OUTPUT);
  pinMode(RIGHT_IN2, OUTPUT);
  stop_motors();
  Serial.begin(9600);
  loadCalibration();
  dumpState();
}
```

- Все шесть пинов драйвера настраиваются как выходы, после чего моторы немедленно останавливаются — платформа не должна тронуться сразу после включения.
- `Serial` поднимается на скорости 9600 бод; этот же аппаратный порт служит каналом связи с Bluetooth-модулем.
- `loadCalibration` восстанавливает калибровку из EEPROM, и `dumpState` выводит её текущее состояние, чтобы пользователь видел стартовые значения.

### loop()

```cpp
void loop() {
  while (Serial.available()) {
    char cmd = Serial.read();
    if (cmd == '\r' || cmd == '\n' || cmd == ' ') continue;
    handleCommand(cmd);
  }
}
```

- На каждой итерации проверяется буфер Serial: все накопленные символы вычитываются и передаются в `handleCommand`.
- Разделители (перевод строки, возврат каретки, пробел) пропускаются, чтобы они не воспринимались как команды.
- Логика обработки событий вынесена в `handleCommand`, поэтому `loop` остаётся компактным и только маршрутизирует входящие символы.

### Обработчик команд handleCommand()

```cpp
void handleCommand(char cmd) {
  switch (cmd) {
    case CMD_X: ... break;
    case CMD_B: ... break;
    case CMD_Y: ... break;
    case CMD_A: ... break;
    ...
  }
}
```

- `handleCommand` — центральная секция, связывающая символ команды с конкретным действием калибровки через `switch`.
- X переключает `currentSide` между левой и правой стороной; последующие команды направления применяются именно к выбранной стороне.
- B инвертирует признак `invertLeft` или `invertRight` в зависимости от выбранной стороны — это перебор способа подключения проводов мотора.
- Y увеличивает `comboIndex` по кругу, A вызывает `applyComboAsForward`, закрепляя текущую комбинацию HIGH/LOW как направление «вперёд».
- Команды крестовины вызывают `changeParam` или переключают `currentParam`; служебные символы запускают проверочные заезды, сохранение в EEPROM, сброс к умолчаниям и вывод состояния.
- Незнакомые символы игнорируются в ветке `default`, что делает приём команд устойчивым к помехам в канале./
